数据分析的价值

不应该只是发现数据中的“经验和常识”，而应当掌握数据探索方法，发现数据潜在的价值，既要遇见“坏的未来”，也要遇见“好的未来”，规避风险的同时抓住机遇，真正体现数据分析工作的价值。

游戏行业：

  • 提供整体运营情况、设立预警指标、定位指标异动原因、同行企业产品对比、发现战略商机、了解同行发展状态、发现市场热点
  • 评估产品质量、帮助产品定位
  • ==对高价值用户进行画像==，分析其行为和偏好、制定营销策略；用户流失预警
  • 分析流失原因、特征，提出版本修正建议
  • ==文本挖掘、分析用户反馈和舆情分析==
  • 监控各位置的转化效率、价值、进行资源位合理安排和定价
  • 帮助开发人员发现问题

数据分析概述

定义

根据“业务理解”，采用“适当的统计方法”对收集的数据进行清洗，提取有用的信息和规律，以简明易懂的方式“呈现”给使用人员，发挥数据的作用。

步骤

  • 明确需求：目的、分析范围、分析时间、分析内容
  • 数据收集：
    • 企业内部数据：游戏行为数据和问卷调查数据
    • 企业外部数据：做舆情监控，竞品分析时，需要从其他渠道收集数据。
  • 数据分析
  • 数据展示

  • 报告撰写：主要采用总分总结构

    • 标题
    • 导语：目的、对象、范围、经过、收获、基本经验等，有侧重点
    • 结论：提供建议进行预测（放在详细分析前面）

    • 详细分析：

      分析报告要层次分明、架构清晰、结论明确、图表化、逻辑性强、对业务了解、不要回避“不良结论”

常用数据分析方法：

  • 对比分析
  • 分组分析
  • 结构分析
  • 平均分析
  • 交叉分析
  • 漏斗分析
  • 矩阵分析
  • 综合评价分析
  • 5W1H：六何分析
  • 相关分析
  • 回归分析
  • 聚类分析
  • 判别分析
  • 主成分分析
  • 因子分析
  • 时间序列分析
  • 方差分析

游戏业务相关数据

游戏关键数据指标

转化率

在手游数据分析中，==需要关注玩家点击广告到进入游戏后付费的每一步转化==，相对于端游，激活率是手游特有的一项数据内容。

激活率

  • 激活、激活率、激活且登录率

    激活：下载安装后，联网登录

    激活率 = 激活量/安装量

    (激活码的)激活率 = 激活量 / 激活码发放量

    (激活码的)激活且登录率 = 激活且登录量 / 激活码发放量

  • 激活且登录率应用场景

    当改转化率较低时，考虑玩家登录是否有困难，排除网络、客户端问题，以及服务器维护等，该指标说明玩家对用户的兴趣程度。不同渠道的数据，反应渠道的用户质量

  • 转化率漏斗

    游戏运行四个主要目标：拉新、促活、留存、付费转化。提高各个环节的转化率

留存率

新用户在一段时间内再次登录游戏的比例。

  • 日留存率：

    • 次日留存率
    • 7日留存率
    • 30日留存率

  • 周留存率：
    新用户在第一周登陆后第二周再次登录的比例

  • 月留存率

  • 加权留存率

    某一时间段内（a）的新增用户在若干天后的另一时间段（b）的留存数量除以之前时间段(a)的新增用户总量。

    采用加权留存率的原因是，当人数变化较大时，数据会产生偏差，加权之后数据更稳定。

用户付费指标

  • PUR（Pay User Rate，付费率）

    PUR = 付费人数/活跃人数

    当付费用户的生命周期总价值有一定的保证，提升付费用户比例，是提升公司营收的有效途径

  • ARPPU（Average Revenue per Paying user）

    ARPPU = 付费金额 / 付费人数

  • ARPU（Average Revenue per User）

    ARPU = 付费金额 / 活跃人数

一般来说，高付费率的游戏，ARPPU比较低，综合来看，ARPU从某种程度上衡量游戏的盈利能力。

导入用户成本

  • CPC（Cost per Click）

    单个点击用户的成本：

    CPC = 广告投入总额/所投的广告带来的点击用户数

  • CPA（Cost per Action）

    每个激活用户的成本：

    CPA = 广告投入总额/所投的广告带来的激活用户数

  • CPR（Cost per Register）

    平均每个注册用户成本：

    CPR = 广告投入总额/所投的广告带来的注册用户数

  • CPR（Cost per Login）

    平均每个登录用户成本：

    CPR = 广告投入总额/所投的广告带来的登录用户数

根据用户转化漏斗情况：CPC < CPA < CPR < CPL ，各项指标的成本越低，说明效果越好，但最终仍要看用户的留存和付费情况，即RIO（Return On Investment ，投资回报率）

LTV(Life Time Value)

指每个用户在生命周期内为该游戏创造的收入总计，可以看做是长期累计的ARPU值，用户的生命周期指用户第一次启动游戏应用到最后一次启动游戏应用之间的周期。

每个用户平均的LTV = 每月ARPU * 用户按月计的平均生命周期

==如何计算用户的平均生命周期?==

LTV指用户在游戏中产出的价值，而CPA是指获取一个有效用户的成本，，当CPA > LTV时，可以理解成获取用户的成本大于用户的产出，通过该数据判断市场投放效果，以及是否有必要追加投放或停止投放尽早止损。

RIO（Return On Investment）

投资回报率：投资后所得的收益与成本间的百分比率

RIO = 利润 / 投资总额 * 100%

  • RIO > 0 : 盈利
  • RIO = 0 ：盈亏平衡
  • RIO < 0 : 亏损

RIO的八大价值：

  1. 衡量产品的盈亏状态；
  2. 筛选推广渠道，分析每个渠道的流量变现能力；
  3. 实时分析，衡量渠道付费流量获取的边际效应，调整投入力度；
  4. 结合其他数据（新增、流失、留存、付费等）调整游戏，进行流量转化与梳理
  5. 综合分析LTV值，对新生产品进行LTV预测，结合CPX衡量推广预算；或综合同类游戏LTV值，进行估值采样，结合CPX衡量推广预算；
  6. 评估后续推广活动的成功与否；
  7. 评估直接ROI即简介ROI的推广优劣；
  8. 推广数据导向，衡量渠道投入性价比。

游戏开发各阶段，数据分析的功能

游戏发行预热期

预定用户分析

  • 对预定用户进行调研，主要步骤为：

    1. 问卷设计
    2. 问卷投放
    3. 数据处理：去除无意义和逻辑互斥数据
    4. 问卷分析
    5. 数据展示
    6. 报告撰写

  • 调研的作用：

    1. 确定宣传渠道
    2. 确定宣传对象
    3. 确定宣传时间
    4. 确定宣传内容
    5. 确定宣传方法

  • 数据来源

    • 用户预定的数据，主要为手机号码和操作平台
    • 用户预定后参与的调查问卷
    • 预定用户在本公司其他游戏中的登录和付费数据

  • 主要的分析指标

    1. 预订用户情况一览：每日预定量、预定渠道分布、预订用户操作系统分布
    2. 预定用户来源：老用户(本公司游戏类型)、新用户（调研常玩游戏）
    3. 预定用户付费：
    4. 预定用户喜好：
    5. 预定用户人口属性：男女比例、年龄分布等
    6. 预定用户地域分布：
    7. 不同类型用户质量：在游戏开启测试后，根据预定用户登录游戏的日志和付费日志，计算用户留存率、付费率和ARPU等关键指标，判断用户质量。

预定用户转化率预估

通过历史其他游戏的预定转化率和该游戏的预订用户量来估计预订用户转化率：

预定用户转化率 = 预定登录游戏 \ 预订用户数量

  • 分析方法：对比分析、分组分析、交叉分析和相关分析
  • 数据来源：用户预定的数据和预定用户在本公司其他游戏的登录和付费数据
  • 分析指标：
    • 用户预订时间分布
    • 预定且登录用户的时间分布
    • 预定且登录用户的登录时间分布

竞品分析

分析内容：

  • 市场宣传、预热活动
  • 开测表现
  • 运行活动及版本计划
  • 数据表现好的原因
  • 畅销榜同类游戏数据对比

详细分析：

  • 测试节点、版本计划、市场活动总览
  • 在ios平台的收入、下载量和排名
  • 进入过畅销榜前50名同类游戏总收入、下载量对比
  • 进入过畅销榜前50名同类游戏总每日收入、下载量对比
  • 版本更新时间表

游戏封测期

问卷调查

  • 用户调查分析：通过用户调查，分析封测的用户人群属性，这些用户对游戏画面、职业、新手引导、技能、副本、社交功能等内容的体验和评价以及建议
  • 渠道质量分析
  • 漏斗分析
  • 留存率评级：根据封测留存率对产品进行评级
  • 数据预测：包含公测市场费、活跃用户、收入、利润率和回收成本
  • 流失分析

数据来源：问卷系统后台数据库

可能涉及的详细分析内容：

  • 调查时间
  • 调查对象和数量
  • 用户整体情况
  • 用户初体验评价
  • 用户人口属性
  • 上手难度
  • 画面与画风
  • 职业
  • 新手引导
  • 地图系统
  • 设计体验
  • 技能
  • 副本
  • 副本组队机制
  • 怪物
  • PVP
  • 武器
  • 装备
  • 社交功能
  • 游戏评价与建议
  • 推荐意愿
